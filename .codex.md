# Crypto Signal Analytic — Codex Context

## Архитектура (коротко)
- **main.py** → `bot.cli.run()`
- **bot/application/bot.py** — runtime orchestration (`SignalBot`)
- **bot/features.py** — Polars-индикаторы, без placeholder-констант
- **bot/setup_base.py** — базовый контракт стратегий
- **bot/strategies/** — конкретные setup-детекторы
- **bot/tracking.py** + **bot/core/memory/repository.py** — трекинг и SQLite/Parquet

## Правила изменений
1. Не вызывать `asyncio.run()` внутри async-кода.
2. Не оставлять placeholder-индикаторы (`pl.lit(0.0/1.0/50.0)`) вместо расчётов.
3. Для стратегий, которым нужны свежие свинги, использовать `_swing_points(..., include_unconfirmed_tail=True)`.
4. При изменении `defaults` в стратегии сверять ключи, читаемые в `detect()`.
5. Для sklearn не передавать Polars напрямую — использовать `.to_pandas()`/`.to_numpy()`.
6. Новые зависимости — только с обновлением `requirements.txt`.

## Частые ловушки
- `depth_imbalance` используется как spread-proxy и не должен проверяться как отрицательный порог для short.
- Для dict использовать `.get(...)`, а не `getattr(...)`.
- Проверять наличие колонок в DataFrame до `item(-1, col)`.
