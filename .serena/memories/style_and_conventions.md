Project conventions: Polars-first, no pandas in new code. Async I/O only; avoid blocking calls in async functions. Strict type hints with `from __future__ import annotations`. Logging via structlog or existing module logger patterns; never print. Strategies inherit from BaseSetup and remain config-driven via config.toml. EventBus is preferred communication pattern. SQLite persistence goes through MemoryRepository / extensions; avoid new JSON persistence. Avoid bare except. Project docs explicitly prefer careful reasoning and honest uncertainty, especially for finance/trading logic.
